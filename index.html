HTML

  • HTML can be broken down into Elements, Tags, and Attributes.
  • The !Doctype tells the browser what type of website to expect - not case sensitive
  • The Head, which contains Metadata that is invisible to end users, is composed of several tags and elements
  • Among these are the elements: Meta, Base, Link, title, and Style
  • Meta can be broken down even further to: Character set, http-equiv, keywords, Viewport, description, and author
  • I've come up with my own acronym for this Kvaud (sounds like a Slavic beverage)
  • Then there's the body, which is made of the bulk of the information on a website
  • Using HTML, you can add pictures to your structure: img src="location" alt="description"
  • There are numbered headers(h1, h2..3..), paragraphs(p), lists(ul/ol/li), sections, breaks(b /), etc.
  • The larger the number next to the h for hearders, the smaller the title/subtitle
  • The paragraph is for normal text in the form of a paragraph. You open the element, enter, then write your text. The you enter and close the element
  • The li is used to create bullets for a bullet point list
  • The ul is to introduce that a list is coming, normal semantic element I think, like: h, img, section etc.
  • We will eventually learn about footers
  • To end an element, you have to put open arrow bracket, slash, tag then close arrow bracket

CSS

  • This is a stylistic language, used to beautify webpages
  • CSS is all about margins, spacing, justification, color, etc.
  • It seems that there will be lots of measurements and technical things there + color
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three main types of CSS: Incline, Internal Stylesheet and Externl Stylesheet
  • The incline is the most cumbersome because you have to add CSS attributes to each and every specific element/tag. Unlike the Stylesheets
  • The other two are added to the head - External Stylesheet is the one mostly used
  • When put in the head, the stylesheets are generalized for the entire page
  • The External is linked to the page in the head using the (Arrow1-link-Arrow2) element
  • That is to say that the CSS External Stylesheet is a separate document with its own code that you link to the site in the head
  • that should look like: **ARROW1-link rel="stylesheet" href="location of stylesheet"ARROW2**
  • "rel" stands for the relationship the following thing will have to the webpage
  • "href" is the reference point, where the information is stored

Git

  • All commands in the CLI that are meant to communicate with GitHub must be prefixed with the git prefix
  • You have to clone a repo from an org's GitHub onto your local enviornment (personal device) to create an IDE using the SSH link section
  • To do so, on the CLI, under the directory you choose, type **git clone SSH Link**
  • A directory feels like a folder to me, and in CLI you can: make them (mkdir _new name_) open files (start _name_), travel between directories (cd _name_) and back one (cd ..)
  • You can also create files by typing in the command line **touch _new name_.txt/docx/ops etc.
  • The command Git checkout -b _new branch name_ creates a new branch for you to work on code from
  • Always work from a new branch as to avoid tampering with main
  • **Git status** tells you where you are in repo, whether your in the main branch or any of the feature branches
  • This point isn't so clear to me. Either you do the the **git add -A** then **git commit -m "Description of what was done"**
  • To ensure that you have up-to-date code, before you launch a PR, put into GitBash command-line **Git pull origin main enter**
  • Once you know that, you can sync your changes org's repo by typing **git push origin _branch name_
